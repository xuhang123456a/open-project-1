# UI 考题

## � 概念题

**Q1: `UIManager` 为什么使用事件驱动而非直接调用？**

<details>
<summary>参考答案</summary>

解耦。UI不需要知道事件来源，只需要响应事件。
</details>

---

**Q2: 暂停时为什么要设置 `Time.timeScale = 0`？**

<details>
<summary>参考答案</summary>

冻结游戏逻辑（物理、动画等），但UI动画仍需使用 `UnscaledDeltaTime`。
</details>

---

**Q3: `ResetUI()` 在什么时候调用？**

<details>
<summary>参考答案</summary>

场景加载完成时（`OnSceneReady`），重置所有面板到初始状态。
</details>

---

**Q4: `FadeController` 为什么要使用 `CanvasGroup`？**

<details>
<summary>参考答案</summary>

`CanvasGroup` 可以统一控制整个UI组的透明度和交互性，无需逐个修改。
</details>

---

**Q5: 打开暂停菜单时为什么要切换输入模式？**

<details>
<summary>参考答案</summary>

暂停时不应响应Gameplay输入（如移动、攻击），只响应菜单输入。
</details>

---

## � 机制题

**Q6: 如果忘记在 `OnDisable` 中取消订阅事件，会有什么问题？**

<details>
<summary>参考答案</summary>

内存泄漏，已销毁对象仍响应事件。
</details>

---

**Q7: `OpenUIPause()` 中为什么要先取消 `MenuPauseEvent` 订阅？**

<details>
<summary>参考答案</summary>

防止暂停菜单打开时再次触发暂停（如按Escape键）。
</details>

---

**Q8: 如果两个面板同时打开，会发生什么？**

<details>
<summary>参考答案</summary>

可能产生冲突。需要面板层级管理或互斥逻辑。
</details>

---

**Q9: `CloseUIPause()` 中为什么要重新订阅 `MenuPauseEvent`？**

<details>
<summary>参考答案</summary>

允许再次打开暂停菜单。
</details>

---

**Q10: 如果 `Time.timeScale = 0` 时UI动画需要播放，应该使用什么？**

<details</summary>

`Time.unscaledDeltaTime`。
</details>

---

**Q11: `LoadingInterfaceController` 的进度是如何更新的？**

<details>
<summary>参考答案</summary>

通过 `SetProgress(float)` 方法，由场景加载协程调用。
</details>

---

## 🔴 架构陷阱题

**Q12: 【同底座差异】`UIManager` 和 `GameStateSO` 都管理状态，有什么区别？**

<details>
<summary>参考答案</summary>

- `UIManager`：UI面板状态
- `GameStateSO`：游戏逻辑状态
</details>

---

**Q13: 【漏步后果】如果 `OpenUIPause()` 忘记设置 `Time.timeScale = 0`，会有什么问题？**

<details>
<summary>参考答案</summary>

游戏逻辑继续运行，玩家可能在暂停时被攻击。
</details>

---

**Q14: 【架构陷阱】如果UI事件回调中执行耗时操作，会有什么问题？**

<details>
<summary>参考答案</summary>

UI卡顿。应将耗时操作放到协程或异步方法中。
</details>

---

**Q15: 【同底座差异】`UIManager` 和 `DialogueManager` 都管理UI，有什么区别？**

<details>
<summary>参考答案</summary>

- `UIManager`：面板显示/隐藏
- `DialogueManager`：对话内容推进
</details>

---

**Q16: 【漏步后果】如果 `ResetUI()` 忘记恢复 `Time.timeScale`，会有什么问题？**

<details>
<summary>参考答案</summary>

游戏可能保持暂停状态。
</details>

---

**Q17: 【架构陷阱】为什么不在每个UI面板上独立订阅事件？**

<details>
<summary>参考答案</summary>

集中管理更易于维护和调试。
</details>

---

**Q18: 【漏步后果】如果 `CloseInventoryScreen()` 忘记触发 `Closed` 事件，会有什么问题？**

<details>
<summary>参考答案</summary>

依赖 `Closed` 事件的系统（如输入模式切换）不会执行。
</details>

## ✍️ 实操题

**Q19: 实现一个 `UIPopup`，显示确认弹窗。**

<details>
<summary>参考答案</summary>

```csharp
public class UIPopup : MonoBehaviour
{
    [SerializeField] private GameObject _panel;
    [SerializeField] private TMPro.TextMeshProUGUI _messageText;
    
    public void Show(string message, UnityEvent onConfirm, UnityEvent onCancel)
    {
        _panel.SetActive(true);
        _messageText.text = message;
        // 绑定按钮事件
    }
    
    public void Hide() => _panel.SetActive(false);
}
```
</details>

---

**Q20 `UITabGroup`，管理标签页切换。**

<details>
<summary>参考答案</summary>

```csharp
public class UITabGroup : MonoBehaviour
{
    [SerializeField] private GameObject[] _tabs;
    
    private int _currentTab;
    
    public void SwitchTab(int index)
    {
        _tabs[_currentTab].SetActive(false);
        _tabs[index].SetActive(true);
        _currentTab = index;
    }
}
```
</details>

---

**Q21: 假设有一个 Bug：暂停菜单打开后无法关闭。请列出至少 3 个可能的原因。**

<details>
<summary>参考答案</summary>

1. `CloseUIPause()` 未被调用
2. 输入模式未切换，按钮事件未响应
3. `Time.timeScale = 0` 导致协程停止
</details>
